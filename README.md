# GEI1092 — Mini Project 2: Sentiment Analysis for Movie Reviews

## Project Overview
This mini-project implements an end-to-end sentiment analysis system for IMDb movie reviews (binary classification: **positive** vs **negative**) using a **deep learning** pipeline:

1. **Text preprocessing** (cleaning, stopword removal, normalization)
2. **Tokenization + sequence padding**
3. **Bidirectional LSTM (BiLSTM)** model for classification

Beyond accuracy, the project addresses **responsible AI** concerns by adding:

- **Carbon footprint tracking** during training and tuning (CodeCarbon)
- **Explainability** with both **LIME** and **SHAP**
- **Deployment considerations** through conversion to **TensorFlow Lite**

## Objectives
- Build a reproducible and documented sentiment analysis workflow on IMDb reviews.
- Train and evaluate a BiLSTM model with standard classification metrics.
- Measure CO2-equivalent emissions generated by training and hyperparameter tuning.
- Interpret individual predictions with local explainability tools (LIME/SHAP).
- Discuss ethical risks (bias, transparency, misuse) in real-world sentiment systems.
- Assess deployment trade-offs by converting the Keras model to TFLite.

## Dataset: Download & Expected Layout
This project uses the **IMDb Large Movie Review Dataset** (Maas et al., 2011).

1. Download: https://ai.stanford.edu/~amaas/data/sentiment/aclImdb_v1.tar.gz
2. Extract the archive.
3. Place the extracted folder at the project root so that the following paths exist:

```
./aclImdb/train/pos
./aclImdb/train/neg
./aclImdb/test/pos
./aclImdb/test/neg
```

Note: the dataset is not committed because it contains **tens of thousands of text files**.

**Citation**
> Maas, A. L., Daly, R. E., Pham, P. T., Huang, D., Ng, A. Y., & Potts, C. (2011). Learning word vectors for sentiment analysis. In *Proceedings of the 49th Annual Meeting of the Association for Computational Linguistics: Human Language Technologies* (pp. 142–150).

## Ethical Considerations & Explainability

### Explainability Method
Because BiLSTM models are less transparent than linear models, this project uses two complementary local explainability tools:

- **LIME**: perturbs text and fits a local surrogate model to identify influential words.
- **SHAP**: computes additive contributions of words/tokens to prediction probabilities.

Using both methods helps verify whether highlighted words are consistent across explainers.

### Ethical Considerations
Key risks and limitations include:

- **Representation bias**: IMDb users are not fully representative of all audiences.
- **Spurious patterns**: the model may learn superficial correlations instead of robust sentiment cues.
- **Language/culture effects**: slang, dialects, and idioms can reduce fairness and reliability.
- **Context limits**: sarcasm, irony, and nuanced sentiment remain difficult.
- **Transparency vs complexity trade-off**: BiLSTM improves context modeling but is harder to audit without explainability tools.

LIME/SHAP improve interpretability, but they do not guarantee that the model is causally correct or unbiased.

## Deployment Considerations (TensorFlow Lite)
The notebook converts the trained Keras model to TensorFlow Lite for lighter deployment:

1. Build a fixed input signature for sequence length 200.
2. Convert model to `.tflite`.
3. Compare model size, inference speed, and accuracy against the original Keras model.

### Observed Outcome
- The converted model is significantly smaller than the Keras artifact.
- Inference is fast enough for constrained environments.
- Accuracy is effectively preserved in the reported experiment.

This demonstrates a practical deployment path with limited performance loss.
