# GEI1092 — Mini Project 1: Sentiment Analysis for Movie Reviews

## Project Overview
This mini-project implements an end-to-end sentiment analysis system for IMDb movie reviews (binary classification: **positive** vs **negative**). The pipeline is intentionally **classical and interpretable**:

1. **Text preprocessing** (cleaning, stopword removal, stemming)
2. **TF-IDF** feature extraction (unigrams + bigrams)
3. **Logistic Regression** classifier

Beyond accuracy, the project addresses **responsible AI** concerns by adding:

- **Carbon footprint tracking** during training and tuning (CodeCarbon)
- **Explainability** via **SHAP** (word-level contributions)
- **Embedded deployment considerations** (fixed-point quantization + export to C++ header)

## Objectives
- Build a reproducible, documented sentiment analysis pipeline.
- Train and evaluate a logistic regression model with standard metrics.
- Quantify energy use and CO2-equivalent emissions produced during training.
- Provide transparent explanations of model decisions using SHAP.
- Discuss ethical and practical deployment risks (bias, misuse, limitations).
- Demonstrate a path toward microcontroller-style inference (quantized coefficients).

## Repository Structure

## Dataset: Download & Expected Layout
This project uses the **IMDb Large Movie Review Dataset** (Maas et al., 2011).

1. Download: https://ai.stanford.edu/~amaas/data/sentiment/aclImdb_v1.tar.gz
2. Extract the archive.
3. Place the extracted folder at the project root so that the following paths exist:

```
./aclImdb/train/pos
./aclImdb/train/neg
./aclImdb/test/pos
./aclImdb/test/neg
```

Note: the dataset is not committed because it contains **tens of thousands of text files**.

**Citation**
> Maas, A. L., Daly, R. E., Pham, P. T., Huang, D., Ng, A. Y., & Potts, C. (2011). Learning word vectors for sentiment analysis. In *Proceedings of the 49th Annual Meeting of the Association for Computational Linguistics: Human Language Technologies* (pp. 142–150).

## Ethical Considerations & Explainability

### Explainability Method
Because logistic regression is linear in the TF-IDF features, word-level contributions can be explained effectively.
The notebook uses:

- `shap.LinearExplainer` with a background sample from the training set
- A SHAP summary plot to visualize the most influential tokens

This helps answer questions like:
- Which words/bigrams drive positive predictions?
- Which terms strongly push the model toward negative sentiment?

### Ethical Considerations
Key risks and limitations for sentiment models on movie reviews include:

- **Representation bias**: IMDb reviewers are not a representative sample of all viewers; the model inherits that distribution.
- **Spurious correlations**: the model can learn shortcuts (e.g., genre-specific terms, writing style) rather than “true” sentiment.
- **Language and culture**: slang, dialects, and cultural references can reduce accuracy and fairness across groups.
- **Context limitations**: bag-of-words approaches do not reliably capture sarcasm/irony (e.g., “great... not”).
- **Accountability**: for real deployment, monitoring, documentation, and periodic re-evaluation are required to avoid silent performance decay.

SHAP does not “prove” correctness, but it improves transparency and can reveal problematic features (e.g., identity terms or sensitive attributes if present).

## Embedded System Deployment Insights
Microcontrollers typically lack floating-point performance and memory to run large Python pipelines. This project demonstrates a practical deployment approach:

1. Train the model in Python.
2. Extract learned parameters (weights and bias).
3. Keep only the **Top-K** most important TF-IDF features.
4. Quantize weights to **fixed-point integers**.
5. Export the resulting model to a C++ header file for inference.

### Exported Artifact: `sentiment_model.h`
The notebook generates `sentiment_model.h` containing:

- `MODEL_W`: quantized weights (`int16_t`)
- `MODEL_B`: quantized bias (`int32_t`)
- `MODEL_FEATURE_NAMES`: the Top-K feature names kept after selection
- Fixed-point inference helpers (integer accumulation + threshold decision)

### Important Practical Note
On embedded hardware, you must still produce an input vector matching the exported features.
In practice, that means either:

- implementing a compatible TF-IDF feature extraction pipeline on-device (often too heavy), or
- computing TF-IDF off-device (PC/phone) and sending only the compact feature vector for inference.
